# 分析方法論

> 涵蓋結構解析、深度分析、批判評估、外部驗證、內化與行動。載入時機：進入完整分析時。
>
> **提醒：本文為 Claude 內部參考。所有方法論名稱（Adler、Zettelkasten、Steel-Manning 等）不出現在使用者可見的報告中。TIPS 評分標準見 SKILL.md。**

## 書型分類

| 書型 | 代碼 | 分析側重 |
|------|------|---------|
| 理論思想 | theory | 論證邏輯、前提假設、可證偽性（Falsifiability） |
| 工具技術 | practical | 步驟可行性、適用範圍、與既有方法的比較 |
| 商業管理 | business | 案例品質、觀點新穎度、可遷移性 |
| 自我成長 | self_help | 科學性特別重要、是否有長期效果的證據 |
| 歷史傳記 | narrative | 史料可靠性、敘事偏見、對當下的啟示 |
| 科普 | pop_science | 是否忠於原始研究、有無過度簡化 |

一本書可橫跨兩個類型，以主要論述為主類型。

## 全書骨架萃取

從目錄和檢視閱讀中萃取邏輯骨架（不是複製貼上目錄）：

1. **核心提問**：作者試圖回答什麼問題
2. **主論點**：作者的回答（一句話）
3. **論證架構**：
   - 線性推演型：A → B → C → 結論
   - 並列佐證型：論點 + 多組證據
   - 問題解方型：問題 → 原因 → 解決方案
   - 對比辯證型：正方 → 反方 → 綜合
4. **概念地圖**：關鍵概念之間的從屬、因果、對比關係

## 深度分析

### Adler 四問 + 樊登四問

先用 Adler 四問建立全貌（談什麼、論點、有道理嗎、跟我什麼關係），再用樊登四問聚焦應用（解決什麼問題、什麼方案、最強論據、怎麼用）。分析過程中自然覆蓋，不需按順序。

關鍵提醒：Adler 第三問「有道理嗎」≠「我同意」。邏輯正確但不接受結論，或直覺認同但論證有洞——兩種判斷要分開處理。

### 底層假設萃取

每本書都建立在作者未明說的假設之上。主動挖掘這些隱含前提：

1. **世界觀假設** — 作者預設什麼樣的世界觀？
2. **讀者假設** — 作者假設讀者是誰？什麼背景？
3. **因果假設** — 作者把什麼當因、什麼當果？有沒有忽略反向因果或共同原因？
4. **文化假設** — 哪些論點只在特定文化脈絡下成立？

底層假設是批判的起點——接受假設則論點成立，拒絕假設則結論動搖。

### 自我解釋（Self-Explanation）

遇到重要段落時用自己的話重述，回答 What / Why / Connection 三問。任一答不出來代表理解不夠。

適合啟動的時機：核心論點、「感覺懂了但說不出來」的段落、中高難度書籍、前後文矛盾處。

### 分析面向（內部思維框架）

> 分析時從五個面向組織思路，再映射到 JSON 對應欄位。

分析過程中區分：**核心論點**（全書 3-7 個）→ core_arguments、**支撐證據** → core_arguments.body、**個人連結**（與讀者經驗的呼應）→ actions、**疑問挑戰**（批判素材）→ critical_perspectives、**方法步驟**（可操作做法）→ actions。

## 批判評估

### Adler 三類反對

只有這三種情況才有資格反對作者：

| 類型 | 判斷標準 | 範例 |
|------|---------|------|
| 知識不足（Uninformed） | 作者沒有考慮到重要事實 | 討論遠端工作卻忽略 2020 年後的實踐數據 |
| 知識錯誤（Misinformed） | 引用的資訊有誤 | 引用的研究後來被撤稿 |
| 邏輯謬誤（Illogical） | 推論有具體的邏輯跳躍 | 用「成功人士都早起」論證「早起導致成功」 |

自我檢查：1) 能具體指出問題嗎？ 2) 有證據支持嗎？ 3) 是反對論證還是反對結論？

### 鋼鐵人論證（Steel-Manning）

批判前先強化作者論點到最好版本：
1. **忠實理解** — 作者看到你的重述會說「對，我就是這個意思」嗎？
2. **主動強化** — 補上最好的論據和最合理的解讀
3. **回應最強版本** — 只有面對最強版本提出的反對才真正有力

### 品質評估維度

| 維度 | 好的跡象 | 壞的跡象 |
|------|---------|---------|
| 論證完整性 | 每個主張都有對應論據 | 多處「顯然」的跳躍 |
| 證據品質 | 一手研究、大樣本 | 依賴軼事、選擇性引用 |
| 內部一致性 | 前後邏輯連貫 | 前面說 A 好，後面推薦非 A |
| 範圍適切性 | 清楚標示適用條件 | 少數案例推論普遍規律 |
| 替代解釋 | 主動回應反面觀點 | 只呈現支持自己的資料 |
| 原創性 | 新框架或新整合 | 換個說法講別人說過的事 |

### 科學性分數與批判深度

S ≤ 1 時：明確標示哪些主張缺乏系統性證據、指出邏輯跳躍、提供替代資訊來源。批判篇幅自然佔 30-40%。

S ≥ 3 時：仍檢視適用範圍和邊界條件、原始研究限制是否被充分揭露。批判篇幅約 15-20%。

## 外部共識驗證（選用）

> 僅在使用者明確要求時執行（例如「幫我查查別人怎麼看這本書」「搜尋書評」）。預設不執行。

### 執行條件

- **使用者明確要求**時才執行，預設跳過
- WebSearch 可用時才執行；若工具不可用，告知使用者並跳過

### 搜尋策略

用 WebSearch 搜尋 2-3 組關鍵字：
1. `「書名」 書評 心得` — 中文社群的討論
2. `"book title" review summary` — 英文社群的評價
3. `「書名」 爭議 批評` — 反面聲音

### 萃取維度

| 維度 | 要回答的問題 |
|------|-----------|
| 公眾記憶 | 這本書在公眾世界真正被記住的是什麼？ |
| 共識觀點 | 哪些觀點被反覆提及、廣泛認同？ |
| 爭議焦點 | 外界最有爭議的部分是什麼？ |
| 常見誤解 | 哪些觀點被大眾過度簡化或誤讀？ |
| 遺漏亮點 | 書中有價值但很少被討論的觀點？ |

外部資料服務於分析，不是取代分析。

## 內化與行動

### Zettelkasten 三層筆記

| 層次 | type 值 | 要求 | 用途 |
|------|---------|------|------|
| 閃念筆記（Fleeting） | `fleeting` | 不限格式，關鍵是有記下來 | 閱讀中即時捕捉 |
| 文獻筆記（Literature） | `literature` | 一則一概念、用自己的話、標註出處 | 整理與轉化 |
| 永久筆記（Permanent） | `permanent` | 完全自己的語言、至少一個連結、說明重要性 | 知識庫核心資產 |

JSON 中的 `zettelkasten[].type` 對應上表的 type 值。每次建立連結時寫一句話說明原因。概念連結是 Zettelkasten 的核心價值。

### 行動承諾三要素

每個行動承諾缺一不可：

| 要素 | 好的範例 | 差的範例 |
|------|---------|---------|
| 時間（When） | 「明天起，每天早上 9 點」 | 「之後有空的時候」 |
| 對象（What context） | 「每週的團隊會議中」 | 「日常工作中」 |
| 具體行動（Specific action） | 「先讓每人各說 2 分鐘再討論」 | 「多傾聽」 |

驗證規則：可觀察、可衡量、可持續（至少兩週）。

完整分析產出 3-5 個行動承諾。一次設定太多反而什麼都做不到，寧可少而精。

## 大型書籍分批合併策略

當書籍需要分塊處理時（estimated_tokens ≥ 80,000），按以下策略合併局部分析：

### 局部筆記格式

每批章節讀完後，以自由格式記錄：
- 該批次的核心論點（含章節來源）
- 新出現的關鍵概念及其定義
- 值得引用的段落（含頁碼）
- 批判觀察（邏輯跳躍、證據不足、與前批矛盾處）
- 可操作的方法步驟

### 合併規則

全部批次讀完後，整合為單一 JSON 時：

1. **去重**：相同概念在不同章節出現時，合併為一個 key_concept，定義取最完整的版本
2. **統一層次**：core_arguments 以全書視角重新組織（不是按章節拼接），通常 3-7 個
3. **跨章批判**：補充只有讀完全書才看得到的矛盾（如前半主張 A，後半卻暗示非 A）
4. **引句精選**：從所有批次中選 3-5 則最有代表性的，避免集中在某一章
5. **行動整合**：去除重複的行動建議，保留最具體可執行的 3-5 個
